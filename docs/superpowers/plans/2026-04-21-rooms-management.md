# План впровадження модуля «Керування аудиторіями» (ASMS V3)

> **Для агентних працівників:** ОБОВ'ЯЗКОВА ПІД-НАВИЧКА: Використовуйте superpowers:subagent-driven-development (рекомендовано) або superpowers:executing-plans для поетапного виконання цього плану. Кроки використовують синтаксис прапорців (`- [ ]`) для відстеження.

**Мета:** Створення повноцінного інтерфейсу для керування аудиторним фондом (CRUD), включаючи облік місткості, обладнання та корпусів.

**Архітектура:** Hilla (React) фронтенд з `@BrowserCallable` Java-ендпоінтами. Використання Tailwind CSS та Aura Theme.

**Технологічний стек:** Java 21, Spring Boot 4, Vaadin 25.1.2 (Hilla), React 19, Tailwind CSS.

---

### Завдання 1: Бекенд — DTO та Endpoint для аудиторій

**Файли:**
- Створити: `src/main/java/com/sergofoox/domain/ui/dto/RoomDTO.java` (якщо не створено або потребує оновлення)
- Створити: `src/main/java/com/sergofoox/domain/room/RoomEndpoint.java`

- [ ] **Крок 1: Оновлення RoomDTO**
  Переконатися, що DTO містить усі необхідні поля: `id`, `name`, `capacity`, `building`, `equipment`, `type`.

- [ ] **Крок 2: Створення RoomEndpoint**
  Реалізувати методи:
  - `List<RoomDTO> getAllRooms()`
  - `void saveRoom(RoomDTO room)`

  - Методи мають бути захищені `@RolesAllowed("DISPATCHER")`.
