# План впровадження модуля «Автоматизація та керування розкладом» (ASMS V3)

> **Для агентних працівників:** ОБОВ'ЯЗКОВА ПІД-НАВИЧКА: Використовуйте superpowers:subagent-driven-development (рекомендовано) або superpowers:executing-plans для поетапного виконання цього плану. Кроки використовують синтаксис прапорців (`- [ ]`) для відстеження.

**Мета:** Реалізація функціоналу автоматичної генерації розкладу за допомогою Timefold Solver та інтерактивного керування на дашборді.

**Архітектура:** Сервісний шар для оркестрації Solver, Hilla ендпоінти для запуску/зупинки, React-інтерфейс для візуалізації прогресу.

**Технологічний стек:** Java 21, Spring Boot 4, Timefold Solver 1.33.0, Vaadin 25.1.2 (Hilla), React 19.

---

### Завдання 1: Бекенд — Сервіс оркестрації Solver

**Файли:**
- Створити: `src/main/java/com/sergofoox/domain/solver/ScheduleService.java`

- [ ] **Крок 1: Реалізація ScheduleService**
  - Метод `solve()`: завантаження всіх сутностей (Teachers, Groups, Rooms, Timeslots, Lessons) у `Schedule`, запуск `SolverManager.solve()`.
  - Обробка результату: збереження знайденого рішення (оновлених `Lesson`) у базу даних.
  - Метод `stopSolving()`: зупинка поточного процесу оптимізації.
  - Метод `getSolverStatus()`: повернення поточного стану (SOLVING, NOT_SOLVING).

---

### Завдання 2: Ендпоінт — Керування життєвим циклом розкладу

**Файли:**
- Модифікувати: `src/main/java/com/sergofoox/domain/ui/ScheduleEndpoint.java`

- [ ] **Крок 1: Додавання методів керування Solver**
  - `void generateSchedule()`: виклик `scheduleService.solve()`.
  - `void clearSchedule()`: видалення всіх призначень (скидання `timeslot` та `room` у всіх `Lesson`).
  - `String getSolverStatus()`: статус для фронтенду.
  - Методи мають бути захищені `@RolesAllowed("DISPATCHER")`.
