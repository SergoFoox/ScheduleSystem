# Дизайн-специфікація: Архівування викладачів

**Дата**: 2026-05-11
**Статус**: Очікує на розгляд
**Мета**: Впровадження механізму "м'якого видалення" викладачів для збереження цілісності історичних даних розкладу та очищення списків призначення від неактивного персоналу.

## 1. Огляд функціоналу

Функція дозволяє переводити викладачів у статус "Архів". Архівовані викладачі:
- Залишаються в існуючих (минулих та поточних) розкладах.
- Приховуються зі списків вибору при призначенні нових занять.
- Відображаються в окремій вкладці інтерфейсу для можливості перегляду або відновлення.

## 2. Технічні деталі (Backend)

### 2.1. Зміни в моделі даних
До сутності `Teacher` додається нове поле:
- `boolean archived` (за замовчуванням `false`).
- Необхідно створити SQL-міграцію `V9__Add_Archived_To_Teacher.sql`.

### 2.2. Логіка репозиторію
У `TeacherRepository` додаються методи фільтрації:
- `List<Teacher> findByArchivedFalse()` — для активних викладачів.
- `List<Teacher> findByArchivedTrue()` — для архіву.

### 2.3. Зміни в Endpoint
- Метод `deleteTeacher(Long id)` змінює поведінку: замість `repository.delete()` він виконує `teacher.setArchived(true)`.
- Додається новий метод `restoreTeacher(Long id)`: встановлює `archived = false`.
- **Фільтрація у списках**: Всі ендпоінти, що повертають списки викладачів для випадаючих меню (наприклад, у `ScheduleEndpoint` або `CoursePlanEndpoint`), повинні повертати лише тих, у кого `archived == false`.

## 3. Дизайн інтерфейсу (Frontend)

### 3.1. Вкладки в розділі "Викладачі"
Використовується компонент `vaadin-tabs`:
- **Вкладка "Активні"**: Відображає стандартну сітку/картки працюючих викладачів та кнопку "Додати викладача".
- **Вкладка "Архів"**: Відображає список архівованих викладачів.

### 3.2. Дії з картками
- Кнопка видалення (кошик) на картці активного викладача ініціює процес архівування.
- На картках в архіві кнопка видалення замінюється на іконку відновлення (`vaadin:rotate-left`).

### 3.3. Візуальні індикатори
- Якщо архівований викладач присутній у сітці розкладу, поруч із його прізвищем відображається маленька іконка 📦 (архів), щоб диспетчер розумів його статус.

## 4. План тестування
- Перевірка, що після архівування викладач зникає з основної вкладки.
- Перевірка, що викладач зникає зі списку "Assign Lesson".
- Перевірка, що викладач залишається у вже створених уроках.
- Перевірка успішного відновлення з архіву.
