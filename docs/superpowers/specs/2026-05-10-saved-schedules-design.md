# Специфікація дизайну: Множинні збережені розклади (Saved Schedules)

**Дата:** 2026-05-10
**Статус:** Схвалено (реалізовано в коді)
**Технологічний стек:** Vaadin 25.1.2 (Hilla/React), Spring Boot 4.0.5, PostgreSQL/H2.

## 1. Огляд
Система підтримує паралельне існування кількох варіантів розкладу. Це дозволяє диспетчеру створювати різні сценарії планування, порівнювати їх та вибирати оптимальний перед публікацією.

## 2. Архітектура та структура даних

### 2.1. Бекенд-сутності
*   **`SavedSchedule`**: Головна сутність, що містить метадані про версію розкладу (назва, дата створення, версія).
*   **`SavedScheduleLesson`**: Спеціальна сутність заняття, прив'язана до конкретного `SavedSchedule`. Вона дублює структуру базового `Lesson`, але має обов'язковий зв'язок `ManyToOne` з розкладом.

### 2.2. Управління життєвим циклом
*   **Каскадне видалення**: Використання `orphanRemoval = true` та анотацій `@OnDelete` на рівні БД для повного очищення занять при видаленні розкладу.
*   **Клонування**: Метод на бекенді для глибокого копіювання існуючого розкладу в новий об'єкт.

## 3. Користувацький інтерфейс

### 3.1. Панель `SavedSchedulesPanel.tsx`
*   Відображається як бічна вкладка або окрема панель.
*   Візуальний індикатор поточного активного розкладу.
*   Кнопки керування: "Створити новий", "Дублювати", "Видалити", "Активувати".

### 3.2. Реактивність
*   Використання глобального сигналу `activeScheduleId` для миттєвого оновлення головної сітки при зміні обраного розкладу.
