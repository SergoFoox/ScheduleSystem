# Специфікація дизайну: Шаблони розкладів (Schedule Templates)

**Дата:** 2026-05-10
**Статус:** Схвалено (реалізовано в коді)
**Технологічний стек:** Vaadin 25.1.2, Spring Boot 4.0.5, Flyway.

## 1. Огляд
Функціонал шаблонів дозволяє зберігати "скелет" розкладу або його повний стан для багаторазового використання. Це мінімізує рутинну роботу диспетчера при плануванні нових семестрів або тижнів.

## 2. Типи шаблонів

### 2.1. Базовий шаблон (Base Template)
*   Зберігає лише структуру: аудиторії та часові слоти.
*   Призначений для швидкого старту нового розкладу на основі існуючих аудиторій.

### 2.2. Повний шаблон (Full Template)
*   Зберігає всі призначення: групи, предмети, викладачі.
*   Використовується для повторюваних тижнів або циклічних планів.

## 3. Технічна реалізація

### 3.1. Міграції БД
*   `V2__Base_Template.sql` та `V3__Saved_Schedule_Full_Template.sql` забезпечують структурну підтримку збереження станів розкладу.
*   Використання окремих таблиць або прапорця `is_template` для ідентифікації шаблонів.

### 3.2. Інтеграція з генератором
*   При запуску Timefold Solver користувач може вказати шаблон як початковий стан ("Seed"), від якого алгоритм почне оптимізацію.
