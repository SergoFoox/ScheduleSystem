# Специфікація дизайну: Обмеження денного навантаження викладача

**Дата:** 2026-05-10
**Статус:** Впровадження (реалізовано)
**Технологічний стек:** Timefold Solver 1.33.0.

## 1. Огляд
З метою запобігання перевтомі викладачів та раціонального розподілу робочого часу, впроваджується глобальне обмеження на кількість занять (пар) в один день для одного викладача. Навіть якщо викладач позначив багато слотів як "Бажані", система намагатиметься не ставити більше 2-х занять на день.

## 2. Логіка обмеження
*   **Цільовий максимум**: 1-2 пари на день.
*   **Жорсткість**: Це м'яке обмеження (Soft constraint), але з високим пріоритетом. Система може поставити 3-ю пару лише у випадку, якщо іншого виходу немає (наприклад, критичний конфлікт груп або аудиторій).
*   **Врахування парності**: Обмеження перевіряється окремо для кожного типу тижня (чисельник/знаменник). Заняття з періодичністю `WEEKLY` враховуються в обох випадках.

## 3. Технічна реалізація
Впроваджено у `ScheduleConstraintProvider.java` через два методи:
*   `teacherMaxLessonsPerDayOdd` — для непарних тижнів.
*   `teacherMaxLessonsPerDayEven` — для парних тижнів.

Використовується групування (`groupBy`) за викладачем та днем тижня з підрахунком кількості через `ConstraintCollectors.count()`. Штраф прогресивно зростає за кожне заняття понад ліміт (2 пари).
