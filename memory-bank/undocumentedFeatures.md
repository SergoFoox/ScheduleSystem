# Недокументовані функції проекту — ASMS V3

Цей документ створено для фіксації функцій та можливостей системи, які були реалізовані в кодовій базі, але не були описані в основних документах (таких як `productContext.md` або `technicalSpecifications.md`).

## 1. Множинні збережені розклади (Saved Schedules)
- **Опис:** Система підтримує створення, збереження та управління кількома різними варіантами розкладу одночасно. 
- **Деталі:** Користувач може створювати нові розклади, перемикатися між ними за допомогою панелі збережених розкладів (`SavedSchedulesPanel.tsx`) та працювати з різними версіями розкладу незалежно.

## 2. Шаблони розкладів (Schedule Templates)
- **Опис:** Реалізовано функціонал створення базових шаблонів розкладу.
- **Деталі:** Дозволяє зберігати поточний стан розкладу як шаблон (`Base Template` або `Saved Schedule Full Template`) і використовувати його як основу для створення нових розкладів у майбутньому.

## 3. Міжрозкладний Drag-and-Drop
- **Опис:** Розширено можливості перетягування (Drag-and-Drop) елементів розкладу.
- **Деталі:** Тепер заняття можна перетягувати не лише в межах одного активного розкладу, але й між різними збереженими розкладами, що значно спрощує процес коригування та перенесення даних.

## 4. Генерація розкладу за курсами (Course Division Tabs)
- **Опис:** Впроваджено вкладки для поділу на курси в процесі генерації.
- **Деталі:** Генератор розкладу тепер може працювати як у режимі створення розкладу для всього навчального закладу загалом, так і для окремих вибраних курсів (наприклад, згенерувати розклад тільки для 1-го та 2-го курсів), що оптимізує продуктивність та локалізує зміни.

## 5. Покращене управління життєвим циклом розкладів
- **Опис:** Виправлено та вдосконалено логіку каскадного видалення.
- **Деталі:** При видаленні розкладу тепер коректно очищаються всі пов'язані з ним заняття та залежності, запобігаючи накопиченню "сміттєвих" даних у базі.

## 6. Локалізація інтерфейсу (labels.ts)
- **Опис:** Впроваджено централізований словник текстових міток.
- **Деталі:** Створено файл `labels.ts` для управління локалізацією та текстами інтерфейсу, що спрощує підтримку різних мов (зокрема, перехід на українську) та зміну термінології без втручання в логіку компонентів.
