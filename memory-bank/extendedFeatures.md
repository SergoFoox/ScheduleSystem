# Додаткові та впроваджені функції — ASMS V3

Цей документ містить опис функціональних та технічних можливостей системи, які були впроваджені під час розробки та які **відсутні або не були детально описані** в початковому [Технічному завданні](technicalSpecifications.md).

---

## 1. Інтелектуальне ядро оптимізації (Timefold Solver)

Окрім базового уникнення конфліктів, впроваджено складні евристики:

*   **Стабільність викладача (`teacherRoomStability`):** Алгоритм мінімізує переміщення викладача між аудиторіями протягом дня. Якщо це можливо, всі пари викладача призначаються в одну "домашню" аудиторію.
*   **Ефективне використання слотів (`compactBiWeekly`):** Система заохочує "парування" занять по чисельнику та знаменнику. Якщо в один час у вівторок є заняття по чисельнику, алгоритм пріоритетно поставить заняття по знаменнику в ту саму комірку.
*   **Рівномірне розсіювання (`spreadRooms`):** Запобігає надмірній концентрації занять в одних аудиторіях, рівномірно розподіляючи навантаження на аудиторний фонд.
*   **Розумний контроль "вікон":** Різні штрафи за розриви в розкладі для студентів та викладачів, що дозволяє створювати максимально компактні графіки.

---

## 2. Модуль "Швидкі заміни" та Коригування

*   **Інтелектуальний підбір кандидатів:** При кліку на заняття система пропонує список `ReplacementCandidateDTO`, де кандидати відсортовані за:
    *   Наявністю компетенції (з Матриці компетенцій).
    *   Поточним тижневим навантаженням (щоб не пропонувати перевантажених колег).
    *   Пріоритетом (основний лектор чи асистент).
*   **Гнучке перепризначення:** Можливість миттєвої зміни предмета, викладача або аудиторії безпосередньо з сітки розкладу.
*   **Логіка підгруп (Split-logic):** Підтримка занять для 1-ї та 2-ї підгруп окремо. Система дозволяє двом викладачам працювати з однією групою одночасно в різних аудиторіях.

## 3. UX та Аналітика в реальному часі

*   **Live Analytics Sidebar:** Бічна панель, що використовує **Signals**, автоматично оновлює дані при будь-якій зміні:
    *   **Підрахунок "вікон":** Миттєва візуалізація кількості розривів для обраного об'єкта.
    *   **Контроль навантаження:** Візуалізація виконання плану (скільки годин вичитано з плану на семестр).
*   **Академічна матриця (PDF-Ready):** Сітка розкладу спроєктована таким чином, що вона ідеально лягає на стандартний формат А3/А4 для друку, включаючи шапку з кураторами та розділення по днях тижня.
*   **Синхронізація періодичності:** Система автоматично перевіряє сумісність періодичності заняття (Чисельник/Знаменник) з типом тижня в обраному часовому слоті.

---

## 4. Технічні та архітектурні рішення

*   **Гібридна стилізація:** Поєднання Tailwind CSS для кастомної сітки та Aura Theme для стандартних компонентів Vaadin.
*   **Data Initializer:** Система автоматичного генерації складних сценаріїв (наприклад, конфліктних ситуацій) для тестування стабільності алгоритмів.
*   **Records-based DTO:** Використання Java Records для всіх обмінів даними, що гарантує високу продуктивність та типобезпеку на фронтенді.
